## 3. STRUKTURA STRONY (mapa serwisu)

### Strona główna `/`
1. **Hero** — krótkie hasło + podtytuł + 2 CTA („Zamów transport" / „Współpraca dla przewoźników") + tło: zdjęcie ciężarówki na drodze (ciemne, kontrastowe, w klimacie księgi znaku)
2. **Pasek zaufania** — np. „11 lat doświadczenia | Spedycja międzynarodowa | Biuro w Poznaniu | Obsługa PL/EN/DE"
3. **O nas (krótko)** — 2–3 zdania + CTA „Poznaj nas"
4. **Usługi** — kafelki: Spedycja międzynarodowa / Transport drogowy / Obsługa dokumentacji / Logistyka kontraktowa (z przypisem „rozwijane")
5. **Dlaczego ES Logistics** — 4–6 punktów (doświadczenie, indywidualne podejście, sieć przewoźników, transparentność, dwujęzyczna obsługa, nowoczesny park — docelowo)
6. **Sekcja dla klientów** — krótki teaser → CTA do formularza „Zamów transport"
7. **Sekcja dla przewoźników** — krótki teaser → CTA do podstrony „Dla przewoźnika"
8. **Liczby / statystyki** — np. 11 lat doświadczenia / X zrealizowanych zleceń / Y krajów / Z przewoźników w sieci (do wypełnienia)
9. **Opinie / referencje** (opcjonalnie — placeholdery jeśli brak)
10. **Sekcja kontaktowa** — adres biura w Poznaniu, telefon, e-mail, mapa Google
11. **Stopka** — dane spółki (NIP, REGON, KRS — do wypełnienia), nawigacja, polityka prywatności, RODO

### Podstrony
- `/o-nas` — historia firmy, sylwetka prezesa, misja, wizja
- `/uslugi` — szczegółowy opis każdej usługi (spedycja międzynarodowa, transport drogowy, dokumentacja, logistyka)
- `/dla-klienta` — proces współpracy w 4 krokach + formularz „Zamów transport" (pola: firma, NIP, e-mail, telefon, miejsce załadunku, miejsce rozładunku, typ ładunku, waga, data, uwagi, RODO)
- `/dla-przewoznika` — sekcja kluczowa, wzór: skat.com.pl/dla-przewoznika
  - Co oferujemy przewoźnikom (stałe zlecenia, terminowe płatności, fair stawki)
  - Wymagania (licencja, OCP, dokumenty)
  - Formularz rejestracji przewoźnika (firma, NIP, flota, kierunki, kontakt, załączniki: licencja + OCP)
- `/kontakt` — formularz ogólny + dane teleadresowe + mapa
- `/polityka-prywatnosci`, `/rodo`, `/regulamin` — wymagane prawnie

- **EN** (obowiązkowa — klienci zagraniczni)

## 7. WYMAGANIA TECHNICZNE

- **Stack (sugestia):** Next.js 14 + TypeScript + Tailwind CSS + shadcn/ui — szybki, SEO-friendly, łatwy w utrzymaniu. Alternatywa: WordPress + Elementor/Bricks jeśli klient potrzebuje samodzielnej edycji.
- **Hosting:** Vercel (jeśli Next) / OVH / cyber_Folks (jeśli WP)
- **Domena:** sugerowana `eslogistics.pl` + `eslogistics.eu` (EN)
- **Responsywność:** mobile-first, breakpointy 360 / 768 / 1024 / 1440
- **Wydajność:** Lighthouse ≥ 90 we wszystkich kategoriach
- **SEO:** meta tagi, Open Graph, schema.org (Organization, LocalBusiness, FAQPage), sitemap.xml, robots.txt
- **Analityka:** Google Analytics 4 + Google Tag Manager + Search Console + Meta Pixel (jeśli kampanie FB/IG)
- **RODO / cookies:** baner zgody (np. Cookiebot lub własny), polityka prywatności, regulamin
- **Formularze:** integracja z e-mailem firmowym + zapis do bazy (np. Supabase / Airtable / własna baza), reCAPTCHA v3, honeypot
- **Mapa:** Google Maps Embed na stronie kontaktowej
- **Bezpieczeństwo:** HTTPS, nagłówki bezpieczeństwa (CSP, HSTS), backup
- **Dostępność (WCAG 2.1 AA):** kontrasty, alt-y, semantyczny HTML, nawigacja klawiaturą
